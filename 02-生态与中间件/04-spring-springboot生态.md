# 04 Spring 与 Spring Boot（零基础版）

## 先用人话说清楚
- Spring 是一个大工厂，帮你管理对象，不用你手写一堆创建和组装代码。
- Spring Boot 是一键装配，帮你快速把项目跑起来。

## 术语翻译
- IoC：对象创建交给容器。
- DI：容器把依赖注入给你。
- AOP：不改业务代码，统一加日志、事务、鉴权。
- 事务：一组操作要么都成功，要么都失败。

## 1. IoC/DI/AOP
### 是什么
- Spring 的三大基础能力：管理对象、注入依赖、织入横切逻辑。

### 为什么
- 可以减少重复代码，让系统更易维护。

### 怎么做
- 业务类只写业务逻辑。
- 通用逻辑交给 AOP 统一处理。

### 高并发如何选择
- 核心接口减少不必要拦截，避免链路太深变慢。

### 面试题与标准答案
- 问：AOP 的价值是什么？
- 答：把日志、事务、权限这类横切逻辑统一管理，减少业务代码污染。

## 2. 事务
### 是什么
- 保证数据操作的一致性。

### 为什么
- 没有事务，扣款成功但订单失败这类问题会大量出现。

### 怎么做
- 事务范围尽量小，避免长事务。
- 跨服务流程用最终一致而不是强行大事务。

### 高并发如何选择
- 核心链路用短事务，减少锁持有时间。

### 面试题与标准答案
- 问：事务为什么会失效？
- 答：常见原因是同类内部调用绕过代理、异常被吞、方法可见性不对。

## 3. Boot 治理
### 是什么
- 自动配置 + 健康检查 + 指标监控。

### 为什么
- 生产环境需要可观测，不然故障不可控。

### 怎么做
- 打开关键指标：线程池、连接池、接口耗时、错误率。

### 高并发如何选择
- 给核心依赖设超时、重试上限、隔离策略。

### 面试题与标准答案
- 问：高并发下 Spring 应用最怕什么？
- 答：线程池耗尽、连接池耗尽、下游慢导致级联超时。

## 本章30秒复述
- Spring 让代码结构更清晰。
- 事务保证一致性但不能滥用。
- 高并发要重点做隔离、超时和监控。

## 追根究底：Spring 背后的代理机制与并发语义

### 1) 为什么同类内部调用会让事务失效
### 是什么
- Spring 事务多数基于代理对象拦截方法调用。

### 为什么
- 同类内部 `this.xxx()` 不走代理，拦截器链不会触发。

### 怎么做
- 把事务边界放在对外公开的方法。
- 避免在同类内部绕过代理调用事务方法。

### 高并发如何选择
- 核心链路务必保证事务入口稳定，减少“看似有注解但实际失效”的风险。

### 面试题与标准答案
- 问：`@Transactional` 为什么有时不生效？
- 答：常见是调用路径没经过代理、异常被吞或方法可见性不满足代理约束。

### 2) 连接池和线程池为什么会联动雪崩
### 是什么
- Web 线程调用数据库连接，连接池耗尽会阻塞业务线程。

### 为什么
- 线程阻塞会占住更多资源，形成“线程等连接、连接等慢 SQL”的放大回路。

### 怎么做
- 为下游调用设置超时上限。
- 监控线程池活跃数、连接池等待时间、慢 SQL。

### 高并发如何选择
- 流量峰值先保护连接池，再谈业务重试；不受控重试会放大故障。

### 面试题与标准答案
- 问：为什么下游慢会拖垮整个 Spring 服务？
- 答：因为线程和连接是有限资源，下游慢导致资源长期占用，上游请求堆积后就会级联超时。

### 3) AOP 链路为什么会增加开销
### 是什么
- 每个切面都是一次方法拦截，链路越深额外开销越大。

### 为什么
- 高频接口上叠加过多切面会增加调用层次、对象创建和上下文切换。

### 怎么做
- 热路径只保留必要切面（鉴权、审计、事务核心逻辑）。
- 非关键逻辑尽量异步化。

### 高并发如何选择
- 在核心接口上“少而精”的切面优于“大而全”的统一拦截。

### 面试题与标准答案
- 问：AOP 是不是越多越好？
- 答：不是。AOP 是治理手段，不是性能优化手段；热路径要控制拦截层级。
