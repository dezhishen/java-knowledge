    Java并发面试题（问题版）¶

    使用说明¶

    • 先独立作答，再对照答案版。
    • 建议按“30秒结论 -> 90秒展开 -> 追问”顺序训练。

    第一题¶

    什么是Java并发中的三大问题（可见性、原子性、有序性）？为什么会出现这些问题？

    作答要求¶

    • 30秒：定义三大问题 + 各自根因
    • 90秒：结合CPU缓存、线程切换、指令重排说明

    深入追问¶

    1. 三大问题里，哪个在工程上最容易引发隐蔽故障？为什么？
    2. JVM/JMM提供了哪些关键手段来解决这些问题？分别解决哪一类问题？

    项目场景追问¶

    请用“并发计数器失真”场景说明问题产生过程，并给出至少两种修复方案。

    第二题¶

    Java线程有哪些状态？Thread.start() 和 Thread.run() 有什么区别？

    作答要求¶

    • 30秒：说全线程状态并指出核心差异
    • 90秒：给出典型状态流转路径

    深入追问¶

    1. BLOCKED和WAITING有什么本质区别？
    2. 如何让线程分别进入WAITING和TIMED_WAITING？

    项目场景追问¶

    以“生产者-消费者”为例，描述线程状态如何在RUNNABLE、WAITING、BLOCKED之间切换。

    第三题¶

    synchronized 和 ReentrantLock 有什么区别？各适用什么场景？

    作答要求¶

    • 30秒：共性 + 关键差异
    • 90秒：从可中断、超时、公平锁、条件变量等维度比较

    深入追问¶

    1. 什么是“可重入”？为什么不可重入可能造成死锁？
    2. 实战中该如何选择synchronized与ReentrantLock？

    项目场景追问¶

    在“缓存热点键防击穿”场景下，如何设计加锁策略兼顾正确性与性能？
