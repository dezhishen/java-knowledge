    Java基础面试题（答案版）¶

    第一题答案¶

    问题：请解释什么是面向对象，以及它解决了什么问题？

    30秒标准回答¶

    面向对象是一种编程范式，把程序要处理的事物抽象成“对象”，每个对象同时包含数据（属性）和行为（方法）。它主要解决大型系统中代码耦合高、难维护、难复用、难扩展的问题。

    90秒展开回答¶

    • 封装：隐藏内部实现，只暴露必要接口，降低耦合。
    • 继承：建立IS-A层次关系，复用已有能力。
    • 多态：统一接口、多种实现，调用方不依赖具体子类。
    • 工程价值：更符合现实建模，更容易扩展新功能，能减少改动范围，降低维护成本。

    2. 封装：如手机缴费，用户只输入号码和金额，不关心后台流程。
    3. 继承：猫是动物，继承动物通用能力。

    4. 多态：同样是“叫”，狗和猫行为不同。

    第二题答案¶

    问题：Java中基本数据类型和String有什么区别？为什么String要设计成不可变？

    30秒标准回答¶

    基本数据类型直接存值；String是引用类型，变量保存对象引用。String不可变是为了性能、安全和并发友好。

    90秒展开回答¶

    String不可变的关键收益： 1. 哈希可缓存：作为HashMap key时性能稳定。 2. 支持字符串常量池：相同字面量可复用，节省内存。 3. 安全性更高：作为参数传递不怕中途被篡改。 4. 天然线程安全：多线程共享无需额外同步。

    第三题答案¶

    问题：Java中函数参数传递是值传递还是引用传递？

    30秒标准回答¶

    Java只有值传递。传对象时，传的是“对象引用的副本”（地址值的拷贝）。

    90秒展开回答¶

    • 传基本类型：拷贝数值，方法内修改不影响外部变量。
    • 传对象引用：拷贝地址值。通过该地址修改对象内容会影响外部可见对象；但把形参重新指向新对象，不会改变外部引用的指向。

    深入追问答案¶

    1. 为什么“改属性会影响外部，改指向不会”： 因为前者是通过同一个对象地址修改对象内容；后者只改了形参这个局部副本的地址值。

    2. 想让外部引用变化怎么办： 让方法返回新对象并由调用方接收；或用可变容器封装后再修改容器内容。

    项目场景参考答案¶

    常见错误：在方法里判断user==null后 user = new User()，调用方引用仍为null，后续可能NPE。修复：返回user并在调用方接收。
