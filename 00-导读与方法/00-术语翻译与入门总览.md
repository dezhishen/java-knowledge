# 00 术语翻译与入门总览

## 这份文件解决什么问题
- 你不需要先懂所有术语。
- 你先把词翻译成生活语言，再去学原理，会轻松很多。

## 高频术语翻译（人话版）
- 并发：很多任务同时来，系统要一起处理。
- 线程：程序里的工人。
- 进程：一个正在运行的程序。
- 锁：同一时间只允许一个工人操作关键资源。
- 原子性：动作要么全成，要么全不成。
- 可见性：一个线程改的数据，另一个线程能及时看到。
- 有序性：代码执行顺序不会乱到影响结果。
- JVM：运行 Java 程序的“发动机”。
- GC：自动回收不用对象的“清洁工”。
- 索引：数据库的目录，帮你快速找数据。
- 事务：一组操作要么一起成功，要么一起失败。
- 缓存：把常用数据放在更快的位置。
- 消息队列：先排队再处理，削峰填谷。
- 限流：系统太忙时先控制进来的人数。
- 熔断：发现下游故障时先断开，避免全局拖死。
- 降级：保核心功能，临时关闭非核心功能。

## 学习地图（从不会到会讲）
1. 第一步：先看术语翻译，能听懂。
2. 第二步：看每章的 是什么/为什么/怎么做。
3. 第三步：背每章“面试题与标准答案”。
4. 第四步：用你自己的项目案例把答案说出来。

## 学习标准（你每天检查）
- 能不能用自己的话讲给外行听。
- 能不能讲清“为什么这样做”。
- 能不能在高并发场景下做选择。
- 能不能回答追问而不慌。
