Java知识网络（从机器约束出发）

这套内容不再以“面试题”为主线，而是以“机器先天约束”作为主线。

核心学习哲学

  • 先看约束：CPU、内存、磁盘、网络、操作系统调度，哪些地方天然慢、天然不可靠。
  • 再看概念：每个技术概念都是为了解决某个具体约束而诞生。
  • 再看取舍：一个方案解决了什么，又引入了什么新问题。
  • 最后看验证：上线后用指标和排障流程验证理解是否正确。

一句话：不是“记住概念”，而是“理解概念为什么存在”。

建议学习路径（按约束来源）

  1. 认知地基：先建立物理机、OS、网络延迟直觉。
  2. Java 平台：语言/集合/并发/JVM 如何应对 CPU 与内存约束。
  3. 数据层：MySQL 如何应对磁盘随机 IO 与一致性约束。
  4. 中间件：Redis/Kafka 如何应对网络延迟与峰值流量。
  5. 分布式：如何在单机能力有限下做扩展与治理。
  6. 工程保障：如何持续观测、定位和恢复线上故障。

每章统一结构（反复出现）

  1. 本章核心约束：到底在解决哪个“先天做不到”。
  2. 物理基础：关键数字与性能直觉。
  3. 概念诞生：概念为何出现、解决了什么。
  4. 边界清单：没解决什么、什么时候失效。
  5. 场景选型：常见方案如何取舍。
  6. 高并发变化：规模放大后如何调整。
  7. 观测排障：看哪些指标、用什么命令排查。

使用方式

  • 导航入口：见 SUMMARY.md
  • 生成规范：见 06-六-训练与固化/03-教材生成流程-固化提示词.md
  • 学习顺序：先读“本章核心约束”和“物理基础”，再读具体技术点。
