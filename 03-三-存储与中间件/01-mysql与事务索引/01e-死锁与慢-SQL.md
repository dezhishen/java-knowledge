## 3. 死锁与慢 SQL
### 是什么
- 死锁是并发锁冲突；慢 SQL 是执行代价过高。

### 为什么
- 二者都直接影响核心接口可用性。

### 怎么做
- 固化死锁排查流程。
- 建慢 SQL 日志和定期治理。

### 高并发如何选择
- 热点写场景优先拆热点，避免集中锁竞争。

### 面试题与标准答案
- 问：遇到死锁先做什么？
- 答：先重试失败事务并收集死锁日志，定位冲突 SQL 和访问顺序。
