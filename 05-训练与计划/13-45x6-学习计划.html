13 45*6学习计划

13 每天45*6学习计划清单

零基础执行法（先听懂再深入）

  • 每个 45 分钟都按这个顺序：
    • 前10分钟：先看 00 文件里的术语翻译。
    • 中25分钟：学习当天主题的人话解释与怎么做。
    • 后10分钟：口述 1 个标准答案（来自 14 文件）。
  • 每天最低目标：
    • 看懂 8-12 个术语。
    • 能完整回答 3 个问题（每题 30-60 秒）。
    • 记录 2 个“我今天终于搞懂”的点。

根因训练法（防止只停留在表面）

  • 每天额外 1 个 45 分钟根因块，固定三步：
    1. 读 15 文件里一个底层点（CPU 或 OS 或数据结构）。
    2. 画一条因果链：现象 -> 根因 -> 证据 -> 方案。
    3. 用 60 秒口述：为什么这个问题会发生。
  • 本周口述检查标准：
    • 不能只说“用某个组件”。
    • 必须说明“为什么这个组件在物理机/OS约束下成立”。

节奏定义

  • 每天 6 个学习块，每块 45 分钟。
  • 推荐结构：2块新知 + 2块强化 + 1块题目 + 1块复盘。

每日固定模板

  1. 第1块（45m）：新知识输入（概念与原理）
  2. 第2块（45m）：新知识输入（机制与源码）
  3. 第3块（45m）：输出训练（口述 是什么/为什么/怎么做）
  4. 第4块（45m）：高并发专项（方案选择与容量估算）
  5. 第5块（45m）：面试题实战（限时答题 + 追问）
  6. 第6块（45m）：复盘与错题整理（沉淀到个人模板）

4周计划（可循环）

第1周：Java并发 + JVM

  • Day1-2：JMM、锁、CAS、线程池。
  • Day3-4：JVM内存、GC、线上排查。
  • Day5-6：并发容器与性能压测。
  • Day7：周复盘 + 模拟面试1次。

第2周：Spring生态 + MySQL

  • Day8-9：Spring 生命周期、事务传播、AOP。
  • Day10-11：MySQL 索引、事务隔离、锁与死锁。
  • Day12-13：SQL 调优、慢查询治理。
  • Day14：周复盘 + 模拟面试1次。

第3周：Redis + Kafka + 分布式

  • Day15-16：缓存模型、一致性与高可用。
  • Day17-18：Kafka 消费模型、重平衡、幂等。
  • Day19-20：分布式事务、限流、熔断、降级。
  • Day21：周复盘 + 模拟面试1次。

第4周：系统设计与综合题

  • Day22-24：高并发场景题（秒杀、订单、支付、库存）。
  • Day25-26：故障演练题（雪崩、热点、消息堆积、数据库抖动）。
  • Day27：项目亮点包装与表达。
  • Day28：全真模拟（2轮技术 + 1轮项目）。

每天交付物清单

  • 1页知识卡（至少3个知识点）。
  • 1份高并发选型对比表（至少2种方案）。
  • 1段3分钟口述录音（用于表达训练）。
  • 1个问题清单（当天未解决问题）。

每日原理与概念学习单（按45*6直接执行）

Day1-7（并发与JVM）

  • Day1：JMM、happens-before、volatile 语义。
  • Day2：synchronized、Lock、AQS、CAS。
  • Day3：线程池参数与拒绝策略原理。
  • Day4：JVM 内存结构与对象生命周期。
  • Day5：GC 算法与 G1 关键参数。
  • Day6：jstack/jmap/arthas 排障方法。
  • Day7：口述复盘：并发 + JVM 体系化串讲。

Day8-14（Spring与MySQL）

  • Day8：IoC/DI/AOP 机制与代理原理。
  • Day9：事务传播、隔离、失效场景。
  • Day10：B+Tree、最左匹配、覆盖索引。
  • Day11：MVCC、Read View、锁模型。
  • Day12：慢 SQL 诊断与执行计划解读。
  • Day13：死锁案例推演与治理策略。
  • Day14：口述复盘：应用框架到数据层的闭环。

Day15-21（Redis/Kafka/分布式）

  • Day15：Redis 数据结构与缓存策略。
  • Day16：穿透、击穿、雪崩与一致性。
  • Day17：Kafka 分区、副本、ISR、位点。
  • Day18：幂等消费、重试、死信。
  • Day19：CAP/BASE 与一致性选型。
  • Day20：分布式锁与配置中心治理。
  • Day21：口述复盘：中间件协同设计。

Day22-28（高并发与系统设计）

  • Day22：容量估算与压测方法。
  • Day23：限流、熔断、降级、背压。
  • Day24：热点治理与削峰填谷。
  • Day25：订单系统一致性与补偿。
  • Day26：秒杀系统防超卖与高可用。
  • Day27：故障演练与应急 SOP。
  • Day28：全真模拟面试与查漏补缺。
